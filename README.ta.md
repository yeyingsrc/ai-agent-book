## 📑 உள்ளடக்க விரைவு அறிமுகம் (அத்தியாயங்கள் 1–10)

முழுப் புத்தகமும் **ஏஜென்ட் = LLM + சூழல் + கருவிகள்** என்ற முக்கிய சூத்திரத்தை மையமாகக் கொண்டு விரிகிறது; பத்து அத்தியாயங்களின் உள்ளடக்கம் பின்வருமாறு:

- **அத்தியாயம் 1 · ஏஜென்ட் அடிப்படைகள்**: "மாதிரியே ஏஜென்ட்" என்ற புதிய முன்னுதாரணத்திலிருந்து தொடங்கி, **ஏஜென்ட் = LLM + சூழல் + கருவிகள்** என்ற முக்கிய சூத்திரத்தை நிறுவுகிறது; மேலும் Harness பொறியியலையும் அறிமுகப்படுத்துகிறது — மாதிரிக்கு அப்பாற்பட்ட அனைத்துப் பொறியியல் திறன்களே உண்மையான போட்டித் திறன் அமைந்த இடமாகும்.
- **அத்தியாயம் 2 · சூழல் பொறியியல்**: சூழல் ஏஜெண்டின் திறனின் மேல் வரம்பைத் தீர்மானிக்கிறது. பெரிய மாதிரி API இன் சூழல் அமைப்பு, KV Cache நட்பு வடிவமைப்பு, விரைவு பொறியியல் (prompt engineering), மாறும் விரைவுகள் மற்றும் Agent Skills, நிலைப் பட்டை மேல்தரவு (metadata), மற்றும் சூழல் சுருக்க உத்திகள் ஆகியவற்றை ஆழமாக ஆராய்கிறது.
- **அத்தியாயம் 3 · பயனர் நினைவகம் மற்றும் அறிவுத் தளம்**: ஏஜென்ட் அமர்வுகளுக்குக் குறுக்கே பயனரை நினைவில் வைத்திருக்கவும், வெளிப்புற அறிவை அணுகவும் செய்கிறது. பயனர் நினைவக அமைப்புகள், RAG அடிப்படைக் குழாய்கள், மற்றும் தட்டையான உரையைத் தாண்டிய அறிவு ஒழுங்கமைப்பு மற்றும் மீட்டெடுப்பு (கட்டமைக்கப்பட்ட குறியீடுகள், அறிவு வரைபடங்கள் போன்றவை) ஆகியவற்றை உள்ளடக்கியது.
- **அத்தியாயம் 4 · கருவிகள்**: கருவிகள் ஏஜெண்டின் கைகளாகும். கருவி வகைப்பாடு மற்றும் பொதுவான வடிவமைப்புக் கொள்கைகள், MCP நெறிமுறை மற்றும் கருவித் தேர்வின் சவால்கள், உணர்தல்/செயலாக்கம்/ஒத்துழைப்பு என்ற மூன்று வகைக் கருவிகள், மற்றும் நிகழ்வு-இயக்கப்படும் ஒத்திசைவற்ற ஏஜென்ட் ஆகியவற்றை விவரிக்கிறது.
- **அத்தியாயம் 5 · Coding Agent மற்றும் குறியீடு உருவாக்கம்**: குறியீடு என்பது "புதிய கருவிகளை உருவாக்கக்கூடிய கருவி" ஆகும், பொது-நோக்க ஏஜெண்டின் மேல்-திறன் (meta-capability) ஆகும். உற்பத்தி-தர Coding Agent ஐ எடுத்துக்காட்டாகக் கொண்டு, இந்த மிகச் சக்திவாய்ந்த பொது-நோக்கக் கருவியின் முழுமையான செயலாக்கத்தைக் காட்டுகிறது.
- **அத்தியாயம் 6 · ஏஜென்ட் மதிப்பீடு**: ஏஜெண்டின் செயல்திறனை ஒப்பிடக்கூடிய சமிக்ஞைகளாக மாற்றுகிறது. மதிப்பீட்டுச் சூழல்கள், தரவுத்தொகுப்பு வடிவமைப்பு, அளவுகோல் அமைப்பு முதல் புள்ளியியல் முக்கியத்துவம், கண்காணிப்புத்தன்மை, மதிப்பீடு-இயக்கப்படும் தேர்வு வரை, மேலும் உற்பத்தி-தர உள் மதிப்பீடு மற்றும் உருவகப்படுத்துதல் சூழல்கள் வரை விவரிக்கிறது.
- **அத்தியாயம் 7 · மாதிரி பிந்தைய பயிற்சி**: முன்-பயிற்சி, SFT, RL என்ற மூன்று நிலைகளின் முழுக் கண்ணோட்டம். எப்போது SFT, எப்போது RL எனத் தேர்வு செய்வது, RLHF, வழிமுறை ஒப்பீடுகள், தரவு மற்றும் சூழல், மேலும் மாதிரி கருவி அழைப்பைக் கற்றுக்கொள்ளச் செய்தல் மற்றும் மாதிரி செயல்திறனை (sample efficiency) மேம்படுத்துதல் போன்ற எல்லை ஆராய்ச்சிகள்.
- **அத்தியாயம் 8 · ஏஜென்ட் சுய-பரிணாமம்**: எடைகளை மாற்றாமலும் வளர முடியும். மூன்று கற்றல் முன்னுதாரணங்கள் — அனுபவத்திலிருந்து கற்றல், செயல்முனைவான கருவி கண்டறிதல், "கருவி பயனரிலிருந்து கருவி படைப்பாளராக மாறுதல்" வரை — ஏஜென்ட்டை "புத்திசாலி" என்பதிலிருந்து "திறன் மிக்க" நிலைக்குக் கொண்டு செல்கிறது.
- **அத்தியாயம் 9 · பல்முக மற்றும் நிகழ்நேர இடைவினை**: உணர்தல் மற்றும் செயலை உரையிலிருந்து குரல், GUI மற்றும் இயற்கை உலகத்திற்கு விரிவுபடுத்துகிறது. குரலின் மூன்று முன்னுதாரணங்கள் (அடுக்கப்பட்ட / இறுதி-முதல்-இறுதி முழு-வடிவ / முழு-இருவழி), ஸ்ட்ரீமிங் குரல் உணர்தல் மற்றும் தொகுப்பு, Computer Use மற்றும் ரோபோ செயல்பாடு ஆகியவற்றை உள்ளடக்கியது.
- **அத்தியாயம் 10 · பல-ஏஜென்ட் கூட்டுச் செயல்பாடு**: குழுவின் நுண்ணறிவு தனிநபரின் நுண்ணறிவை விட உயர்ந்ததாக இருக்கலாம். பல-ஏஜென்ட் வகைப்பாட்டுக் கட்டமைப்பு, எப்போது உண்மையில் ஒற்றை ஏஜென்ட்டை விடச் சிறந்தது, சூழலைப் பகிர்ந்து கொள்ளும் மற்றும் பகிராத ஒத்துழைப்பு, தோல்வி முறைகள், மற்றும் தோன்றும் "ஏஜென்ட் சமூகம்" ஆகியவற்றை விவரிக்கிறது.

## 💻 இணைக் குறியீடு

அனைத்துத் திட்டங்களும் **அத்தியாயம்** வாரியாக ஒழுங்கமைக்கப்பட்டுள்ளன, புத்தகத்தின் பத்து அத்தியாயங்களுடன் ஒன்றுக்கொன்று ஒத்திசைகின்றன, அடிப்படைக் கருத்துக்களிலிருந்து மேம்பட்ட நுட்பங்கள் வரையிலான முழுமையான கற்றல் பாதையை உள்ளடக்கியது; கோப்பக அமைப்பு `chapterN/திட்டம்-பெயர்/` வடிவில் உள்ளது. அத்தியாயங்கள் 5, 8, 9, 10 இல் உள்ள பெரும்பாலான சோதனைகளுக்கு இப்போது சுயாதீனமாக இயக்கக்கூடிய இணை demo வழங்கப்படுகின்றன, மேலும் அவை உண்மையான LLM API உடன் இணைத்துச் சரிபார்க்கப்பட்டு இயக்கப்பட்டுள்ளன.

3 வகை தாக்குதல் சூழ்நிலைகள் (நேரடி செலுத்துதல், மறைமுக செலுத்துதல், நினைவக செலுத்துதல்) × 4 வகை பாதுகாப்பு உள்ளமைவுகள் (பாதுகாப்பு இல்லை, வழிகாட்டி வலுப்படுத்தல், மூலக் குறிப்பு, இணைந்த பாதுகாப்பு) கொண்ட ஒப்பீட்டுப் பரிசோதனைகளை உருவாக்கி, தீர்மான விதிகளால் தாக்குதல் வெற்றி விகிதத்தைக் கணக்கிட்டு, அடுக்கடுக்காகப் பாதுகாப்பைச் சேர்த்த பிறகு செலுத்துதல் வெற்றி விகிதம் எவ்வாறு கணிசமாகக் குறைகிறது என்பதை நேரடியாகக் காட்டுகிறது.

**முக்கிய கருத்துக்கள்**: வழிகாட்டி செலுத்துதல், மறைமுக செலுத்துதல், தரவு-வழிமுறை பிரிப்பு, இயக்கநேர சரிபார்ப்பு

### agent-skills-ppt - Agent Skills படிப்படியான வெளிப்படுத்தல் மூலம் PPT உருவாக்கம்
`chapter2/agent-skills-ppt/`

Agent Skills இன் "படிப்படியான வெளிப்படுத்தல்" (progressive disclosure) கருத்தை மீண்டும் உருவாக்குகிறது: ஏஜென்ட் தொடங்கும்போது ஒரு மெல்லிய Skill அட்டவணையை மட்டுமே காண்கிறது; பணிக்கு `pptx` Skill தேவை என்பதை அடையாளம் கண்ட பிறகே அதன் முழுமையான செயல்முறை, விவரமான ஆவணங்கள் மற்றும் இணைக்கப்பட்ட ஸ்கிரிப்டுகளை அடுக்கடுக்காக ஏற்றி, இறுதியாக python-pptx மூலம் உண்மையான `.pptx` கோப்பை உருவாக்குகிறது.

**முக்கிய கருத்துக்கள்**: Agent Skills, படிப்படியான வெளிப்படுத்தல், தேவைக்கேற்ப ஏற்றுதல், கருவி ஒருங்கிணைப்பு

### user-memory - பயனர் நினைவக அமைப்பு
`chapter3/user-memory/`

நீண்டகால பயனர் நினைவக அமைப்பை உருவாக்குதல், ஏஜென்ட் பயனர் விருப்பங்களையும் கடந்தகால தொடர்புகளையும் நினைவில் வைத்து தனிப்பயனாக்கப்பட்ட சேவையை வழங்க உதவுகிறது.

**முக்கிய கருத்துக்கள்**: நீண்டகால நினைவகம், தனிப்பயனாக்கம், பயனர் மாதிரியமைப்பு

### mem0 / memobase - திறந்த மூல நினைவகக் கட்டமைப்பங்களின் ஒப்பீடு
`chapter3/mem0/` மற்றும் `chapter3/memobase/`

mem0 மற்றும் Memobase ஆகிய இரண்டு திறந்த மூல நினைவகக் கட்டமைப்புகளில் ஒவ்வொன்றிலும் பயனர் நினைவகத்தின் ஒரு பதிப்பைச் செயல்படுத்துகிறது; இது சோதனை 3-2 "நினைவக உத்திகளின் ஒப்பீடு" இன் ஒப்பீட்டுச் செயலாக்கமாகச் செயல்பட்டு, வெவ்வேறு நினைவகத் தீர்வுகளின் பிரித்தெடுப்பு வடிவங்களையும் பதில் தரத்தையும் குறுக்காக ஒப்பிடுவதை எளிதாக்குகிறது.

**முக்கிய கருத்துக்கள்**: நினைவகக் கட்டமைப்பு, mem0, Memobase, தீர்வுகளின் ஒப்பீடு

### user-memory-evaluation - பயனர் நினைவக மதிப்பீட்டுக் கட்டமைப்பு
`chapter3/user-memory-evaluation/`

பயனர் நினைவக அமைப்புகளின் துல்லியம், தொடர்பு மற்றும் செயல்திறன் ஆகியவற்றை முறையாக மதிப்பீடு செய்கிறது; பல சோதனைச் சூழல்களையும் மதிப்பீட்டு அளவுகோல்களையும் உள்ளடக்கியது.

**முக்கிய கருத்துக்கள்**: மதிப்பீட்டுக் கட்டமைப்பு, சோதனை வழக்குகள், செயல்திறன் அளவீடுகள்

### dense-embedding - அடர்த்தியான உட்பொதிப்பு திசையன் மீட்டெடுப்புச் சேவை
`chapter3/dense-embedding/`

திசையன் ஒற்றுமை தேடல் சேவையை உருவாக்கி, ANNOY (மர அடிப்படையிலான) மற்றும் HNSW (வரைபட அடிப்படையிலான) ஆகிய இரண்டு ANN குறியீட்டு வழிமுறைகளை ஒப்பீட்டு ஆராய்கிறது. வெவ்வேறு குறியீட்டு உத்திகளின் செயல்திறன், நினைவக பயன்பாடு மற்றும் புதுப்பிப்புத் திறன் ஆகியவற்றிலுள்ள சமநிலைகளை (trade-offs) விளக்குகிறது.

**முக்கிய கருத்துக்கள்**: அடர்த்தியான உட்பொதிப்பு, திசையன் மீட்டெடுப்பு, ANN வழிமுறை, சொற்பொருள் தேடல்

### sparse-embedding - அரிதான மீட்டெடுப்புப் பொறி
`chapter3/sparse-embedding/`

BM25 வழிமுறையை அடிப்படையாகக் கொண்ட அரிதான திசையன் தேடல் பொறியை அடிப்படையிலிருந்து செயல்படுத்துகிறது; வளமான பதிவுகள் மற்றும் காட்சிப்படுத்தல் இடைமுகங்கள் மூலம் தேடல் பொறியின் உள் செயல்பாட்டு வழிமுறையை விளக்கி, சொல் அதிர்வெண் எடை கணக்கீடு மற்றும் தலைகீழ் குறியீட்டு (inverted index) கொள்கைகளைப் புரிந்து கொள்ள உதவுகிறது.

**முக்கிய கருத்துக்கள்**: அரிதான உட்பொதிப்பு, BM25, TF-IDF, துல்லிய பொருத்தம்

### retrieval-pipeline - கலப்பின மீட்டெடுப்புக் குழாய்
`chapter3/retrieval-pipeline/`

அடர்த்தியான மீட்டெடுப்பு, அரிதான மீட்டெடுப்பு மற்றும் நரம்பியல் மறு-தரவரிசைப்படுத்தல் ஆகியவற்றை இணைத்து ஒரு முழுமையான மீட்டெடுப்புக் குழாயை உருவாக்குகிறது. கவனமாக வடிவமைக்கப்பட்ட சோதனை வழக்குகள் மூலம், வெவ்வேறு சூழல்களில் கலப்பின மீட்டெடுப்பின் பரஸ்பர நிரப்பு நன்மைகளை முறையாக விளக்குகிறது.

**முக்கிய கருத்துக்கள்**: கலப்பின மீட்டெடுப்பு, நரம்பியல் மறு-தரவரிசைப்படுத்தல், கிராஸ்-என்கோடர் (Cross-Encoder), மீட்டெடுப்பு இணைவு

### multimodal-agent - பல்லூடக தகவல் பிரித்தெடுத்தல்
`chapter3/multimodal-agent/`

மூன்று பல்லூடக செயலாக்க உத்திகளை ஒப்பிடுகிறது: சொந்த பல்லூடக செயலாக்கம், உரையாகப் பிரித்தெடுத்தல், கருவி-அடிப்படையிலான பகுப்பாய்வு. ஒருங்கிணைந்த கட்டமைப்பின் கீழ் நடத்தப்படும் நீக்கல் ஆய்வுகள் (ablation studies) மூலம், வெவ்வேறு தொழில்நுட்பப் பாதைகளின் உண்மைத்தன்மை (fidelity), செலவு மற்றும் நெகிழ்வுத்தன்மை ஆகியவற்றிலுள்ள சமநிலைகளை வெளிப்படுத்துகிறது.

**முக்கிய கருத்துக்கள்**: பல்லூடக, காட்சி புரிதல், OCR, இறுதி-முதல்-இறுதி செயலாக்கம்

### structured-index - கட்டமைக்கப்பட்ட குறியீடு
`chapter3/structured-index/`

RAPTOR (மீள்சுருக்க மரம்) மற்றும் GraphRAG (அறிவு வரைபடம்) ஆகிய இரண்டு மேம்பட்ட குறியீட்டு உத்திகளைச் செயல்படுத்தி ஒப்பிடுகிறது. ஒரு குறியீட்டு தொழில்நுட்பக் கையேட்டின் மூலம், அறிவின் உள்ளார்ந்த படிநிலை மற்றும் தொடர்புகளைப் பிரதிபலிக்கும் கட்டமைக்கப்பட்ட குறியீடுகளை எவ்வாறு உருவாக்குவது என்பதை விளக்குகிறது.

**முக்கிய கருத்துக்கள்**: RAPTOR, GraphRAG, படிநிலை சுருக்கம், அறிவு வரைபடம்

### agentic-rag - Agentic RAG
`chapter3/agentic-rag/`

பாரம்பரிய Non-Agentic RAG மற்றும் Agentic RAG ஆகியவற்றின் செயல்திறன் வேறுபாடுகளை ஒப்பிடுகிறது. ReAct முறையின் மூலம் ஏஜென்ட் சுழற்சிமுறை தகவல் மீட்டெடுப்பை எவ்வாறு வழிநடத்துகிறது என்பதை விளக்கி, சிக்கலான நீதித்துறை கேள்வி-பதில்களைக் கையாளும்போது பதில் தரத்தை கணிசமாக மேம்படுத்துகிறது.

**முக்கிய கருத்துக்கள்**: Agentic RAG, ReAct சுழற்சி, சுழற்சிமுறை மீட்டெடுப்பு, செயலூக்க ஆராய்தல்

### agentic-rag-for-user-memory - Agentic RAG ஐப் பயன்படுத்தி பயனர் நினைவகத்தை உருவாக்குதல்
`chapter3/agentic-rag-for-user-memory/`

Agentic RAG கட்டமைப்பை பயனர் உரையாடல் வரலாற்றை நிர்வகிப்பதற்குப் பயன்படுத்துகிறது; பல-திருப்ப மீள்தேடல் திறன் மூலம் குறுக்கு-அமர்வு நினைவக மீட்டெடுப்பைக் கையாண்டு, அடிப்படை நினைவுகூரல் மற்றும் பல-அமர்வு மீட்டெடுப்புத் திறன்களைச் செயல்படுத்துகிறது.

**முக்கிய கருத்துக்கள்**: பயனர் நினைவகம், உரையாடல் வரலாற்று குறியீட்டமைப்பு, குறுக்கு-அமர்வு மீட்டெடுப்பு

### contextual-retrieval - சூழல்-உணர்வு மீட்டெடுப்பு
`chapter3/contextual-retrieval/`

Anthropic முன்மொழிந்த சூழல்-உணர்வு மீட்டெடுப்பு (context-aware retrieval) நுட்பத்தைச் செயல்படுத்துகிறது; உரைத் துண்டுகளுக்கு முக்கிய சூழலைக் கொண்ட முன்னொட்டுச் சுருக்கங்களை உருவாக்குவதன் மூலம், பாரம்பரிய துண்டாக்கும் முறைகளின் சூழல் இழப்புச் சிக்கலைத் தீர்த்து, மீட்டெடுப்புத் தோல்வி விகிதத்தை 49-67% குறைக்கிறது.

**முக்கிய கருத்துக்கள்**: சூழல் மேம்படுத்தல், முன்னொட்டு உருவாக்கம், சொற்பொருள் நங்கூரமிடல், மீட்டெடுப்பு மேம்படுத்தல்

### contextual-retrieval-for-user-memory - சூழல்-உணர்வு பயனர் நினைவக அமைப்பு
`chapter3/contextual-retrieval-for-user-memory/`

சூழல்-உணர்வு மீட்டெடுப்பு நுட்பத்தை பயனர் நினைவக உருவாக்கத்திற்குப் பயன்படுத்தி, Advanced JSON Cards ஐ சூழல்-உணர்வு RAG உடன் இணைத்து இரண்டு-அடுக்கு நினைவக கட்டமைப்பை உருவாக்கி, மேலும் உயர்ந்த மட்ட செயலூக்க சேவைத் திறனைச் செயல்படுத்துகிறது.

**முக்கிய கருத்துக்கள்**: இரண்டு-அடுக்கு நினைவகம், கட்டமைக்கப்பட்ட உண்மைகள், சூழல் சார்ந்த மீட்டெடுப்பு, செயலூக்க சேவை

### structured-knowledge-extraction - கட்டமைக்கப்பட்ட அறிவு பிரித்தெடுத்தல்
`chapter3/structured-knowledge-extraction/`

நீதித்துறைத் தீர்ப்பு வழக்குகளை உதாரணமாகக் கொண்டு "கீழிருந்து மேல் காரணி கண்டுபிடிப்பு → வழக்கு முன்மாதிரிகளாக கிளஸ்டரிங் → உரையாடல் அறிவுரை ஏஜென்ட்" என்ற மூன்று-கட்டக் குழாயை முழுமையாக இயக்குகிறது: முன்கூட்டியே வரையறுக்கப்பட்ட கடினமான புலங்கள் இல்லாமல், LLM ஏராளமான வழக்குகளிலிருந்து தானாகவே காரணிகளைக் கண்டுபிடித்து அவற்றை ஒரு தொகுதி schema வாக (மையக் காரணிகள் + குற்றச்சாட்டு-சார்ந்த நீட்டிப்புக் காரணிகள்) தொகுக்கிறது; பின்னர் வழக்குகள் பல முன்மாதிரிகளாகக் கிளஸ்டர் செய்யப்பட்டு, ஒவ்வொரு முன்மாதிரியின் காரணி முக்கியத்துவமும் கணக்கிடப்படுகிறது; ஏஜென்ட் புதிய வழக்கின் விவரங்களுக்கு மிகவும் ஒத்த முன்மாதிரியைப் பொருத்தி, முக்கியத்துவத்தின் அடிப்படையில் விடுபட்ட தகவல்களைப் பின்தொடர்ந்து கேட்டு, ஆதாரமுள்ள அறிவுரையை வழங்குகிறது (சட்ட மறுப்புரை இணைக்கப்பட்டுள்ளது).

**முக்கிய கருத்துக்கள்**: கீழிருந்து மேல் அறிவு கண்டுபிடிப்பு, தொகுதி காரணிகள், கிளஸ்டரிங் முன்மாதிரிகள், விளக்கக்கூடிய முடிவெடுப்பு

### async-agent - இணையான செயலாக்கம் மற்றும் குறுக்கிடும் திறன் கொண்ட ஒத்திசையற்ற ஏஜென்ட்
`chapter4/async-agent/`

asyncio ஒற்றை-நூல் அடிப்படையில் நிகழ்வு-உந்துதல் ஒத்திசையற்ற ஏஜென்ட் கட்டமைப்பின் (Flux) மையத்தைச் செயல்படுத்துகிறது: inbox நிகழ்வு வரிசை அவசர நிலையின்படி ஒதுக்குகிறது (குறுக்கிடல்/உடனடி/வரிசையில்), ஒத்திசையற்ற கருவிகளின் இணையான செயலாக்கத்தை ஆதரித்து, இயங்கும் போது தற்போதைய turn-ஐ குறுக்கிட அனுமதித்து, உருவகப்படுத்தப்பட்ட நீண்ட பணிகளுக்கு ரத்து செய்தல் மற்றும் நிலை வினவலையும் வழங்குகிறது. முடிவெடுப்பது உண்மையான LLM (function calling) மூலம் நிறைவேற்றப்படுகிறது.

**முக்கிய கருத்துக்கள்**: ஒத்திசையற்ற நிரலாக்கம், நிகழ்வு வரிசை, குறுக்கிடும் வழிமுறை, இணையான கருவி ரத்து, தடையற்ற I/O

### coding-agent - உற்பத்தி-தர Coding Agent
`chapter5/coding-agent/`

Claude-இல் கட்டப்பட்ட உற்பத்தி-தர AI குறியீட்டு உதவியாளர்; அனைத்துக் கருவிகளையும் தூய Python-இல் செயல்படுத்துகிறது, கட்டளை-வரிச் சார்புகள் எதுவும் தேவையில்லை. கோப்புச் செயல்பாடுகள், தேடல், Shell செயல்பாடுகள் மற்றும் திட்ட நிர்வாகம் உள்ளிட்ட 17 முழுமையாகச் செயல்படுத்தப்பட்ட கருவிகளைக் கொண்டுள்ளது. ripgrep-இன் செயல்பாடுகளுடன் முழுமையாகப் பொருந்தக்கூடிய தூய Python Grep கருவியையும் குறிப்பாகச் செயல்படுத்துகிறது.

**முக்கிய அம்சங்கள்**:
- தூய Python செயலாக்கம், கட்டளை-வரிச் சார்புகள் இல்லை, குறிப்பாக Mac பயனர்களுக்கு ஏற்றது

- கணினி அறிவுறுத்தல் (system prompt) நுட்பங்கள்: நேர முத்திரைகள், கருவி அழைப்பு எண்ணிக்கை, TODO பட்டியல் நிர்வாகம், விரிவான பிழைச் செய்திகள்
- நிலைத்திருக்கும் Shell சூழல், தானியங்கு Lint கண்டறிதல், ஓடை (streaming) பதில் ஆதரவு
- பல LLM வழங்குநர்களுக்கான ஆதரவு (Anthropic, OpenAI, OpenRouter)

**முக்கிய கருத்துக்கள்**: குறியீடு உருவாக்கம், கோப்புத் திருத்தம், தூய Python கருவிகள், கணினி அறிவுறுத்தல், Lint கண்டறிதல், பல-வழங்குநர் ஆதரவு

அதே மாதிரியை அதே போட்டிக் கணிதக் கணக்குகள் தொகுப்பில் "தூய சிந்தனைச் சங்கிலி" மற்றும் "குறியீடு-உதவி" என்ற இரண்டு முறைகளில் ஒப்பிடுகிறது: பிந்தைய முறையில், கணக்குகள் Python-ஆக (sympy/numpy/scipy) முறைப்படுத்தப்பட்டு, function calling மூலம் துணை-செயல்முறை மணல் பெட்டியில் இயக்கப்படுகின்றன; பிழைக்கு உட்பட்ட மனக்கணக்குக்குப் பதிலாகத் துல்லியமான கணக்கீடு பயன்படுத்தப்படுவதால், துல்லியத்தன்மை குறிப்பிடத்தக்க அளவில் அதிகமாக உள்ளது.

**முக்கிய கருத்துக்கள்**: குறியீடு விளக்கி, குறியீட்டுக் கணிதம் (symbolic computation), சிந்தனைச் சங்கிலி ஒப்பீடு, கருவி-மேம்படுத்தப்பட்ட பகுத்தறிவு

### code-for-logic - குறியீட்டின் மூலம் தர்க்கச் சிந்தனைத் திறனை மேம்படுத்துதல்
`chapter5/code-for-logic/`

"Knights and Knaves" தர்க்கப் புதிர்களைக் கட்டுப்பாடு திருப்திக் கணக்குகளாக (CSP) மாற்றுகிறது: ஏஜென்ட் `python-constraint` ஐப் பயன்படுத்தி மாறிகள் மற்றும் இரு-நிபந்தனைக் கட்டுப்பாடுகளை வரையறுத்துத் தீர்வியை அழைக்கிறது; ஒரு K&K புதிர்த் தொகுப்பில் தூய இயற்கை மொழிப் பகுத்தறிவு மற்றும் குறியீடு-உதவி என்ற இரு முறைகளின் துல்லியத்தன்மையை ஒப்பிடுகிறது.

**முக்கிய கருத்துக்கள்**: கட்டுப்பாடு தீர்வு, CSP மாதிரியாக்கம், முறைப்படுத்தப்பட்ட பகுத்தறிவு, குறியீடு உதவி

### small-model-codified-rules - சிறிய மாதிரிகளுக்கான குறியீடாக்கப்பட்ட விதிகள்
`chapter5/small-model-codified-rules/`

τ-bench விமான வாடிக்கையாளர் சேவைச் சூழ்நிலையை அடிப்படையாகக் கொண்ட கட்டுப்பாட்டுச் சோதனை: சிக்கலான வணிகக் கொள்கைகளை (திருப்பிச் செலுத்தும் விதிகள்) இயற்கை மொழி வரியூட்டல்களிலிருந்து குறியீடு/கருவிகளுக்குள் நகர்த்திய பிறகு, சிறிய மாதிரியின் பணி வெற்றி விகிதமும் கொள்கை இசைவும் பெரிதும் உயர்ந்தன; கருவிக்குள் உள்ள குறியீடுச் சரிபார்ப்பு மாதிரியின் தவறான புரிதல்களை நிகழ்நேரத்தில் தடுக்கிறது.

**முக்கிய கருத்துக்கள்**: குறியீடாக்கப்பட்ட வணிக விதிகள், கொள்கை நடைமுறைப்படுத்தல், கருவிக்குள் சரிபார்ப்பு, சிறிய மாதிரி நம்பகத்தன்மை

### paper-to-ppt - ஆய்வுக்கட்டுரையிலிருந்து தானியங்கு PPT உருவாக்கம் (முன்மொழிபவர்-மதிப்பாய்வாளர்)
`chapter5/paper-to-ppt/`

"PPT உருவாக்குதல்" என்பதைக் குறியீடு உருவாக்கக் கணக்காக மறுசீரமைக்கிறது: Proposer Slidev (Markdown+HTML) குறியீட்டை எழுதுகிறது, Reviewer ஒவ்வொரு பக்கத்தையும் உண்மையில் PNG-ஆக வழங்கி (render), Vision LLM மூலம் வடிவமைப்புப் பிரச்சினைகளைச் சரிபார்த்து, கட்டமைக்கப்பட்ட பின்னூட்டத்தின் அடிப்படையில் மீள்திருத்தம் செய்கிறது; இரட்டை ஏஜென்ட் பணிப்பிரிவு சூழலின் உச்ச அளவைக் குறிப்பிடத்தக்க வகையில் குறைக்கிறது.

**முக்கிய கருத்துக்கள்**: குறியீடு உருவாக்கம், Slidev, முன்மொழிபவர்-மதிப்பாய்வாளர், காட்சித் தரக் கட்டுப்பாடு

### paper-to-video - ஆய்வுக்கட்டுரை விளக்கக் காணொளி தானியங்கு உருவாக்கம்
`chapter5/paper-to-video/`

"ஆய்வுக்கட்டுரை → PPT" என்பதன் அடிப்படையில் ஒவ்வொரு ஸ்லைடுக்கும் பேச்சுவழக்கான விளக்க உரையை உருவாக்கி, TTS ஐ அழைத்துக் குரலைத் தொகுத்து, பின்னர் ffmpeg ஐப் பயன்படுத்தி ஒவ்வொரு பக்கத்தின் திரைப்பிடிப்பையும் அதன் ஒலியையும் பக்கம் தோறும் ஒத்திசைத்து, விவரிப்புடன் கூடிய ஒரு விளக்கக் காணொளியாக இணைக்கிறது.

**முக்கிய கருத்துக்கள்**: பல்லூடக உருவாக்கம், விளக்க உரை உருவாக்கம், TTS, ffmpeg ஒலி-காணொளி ஒத்திசைவு

### video-edit - API-அடிப்படையிலான நுண்ணறிவுக் காணொளி தொகுப்பு
`chapter5/video-edit/`

பயனர் ஒரு பல-காட்சிக் காணொளியையும் ஒரு இயற்கை மொழித் தேவையையும் தர, ஏஜென்ட் "இரு-படி Vision இருப்பிடக் கண்டறிதல்" (முதலில் தோராயமாகவும் பின்னர் நுணுக்கமாகவும் சட்டகங்களை எடுத்துப் படித்தல்) மூலம் இலக்குக் காட்சியின் நேர எல்லைகளைத் தீர்மானிக்கிறது; துண்டை வெட்டிய பிறகு Reviewer இறுதிக் காணொளியிலிருந்து முக்கியச் சட்டகங்களை எடுத்துச் சரிபார்க்கிறது, தரமில்லையென்றால் மீள்செய்கிறது.

### adaptive-log-parser - தழுவல் பதிவு பாகுபடுத்தல் அமைப்பு
`chapter5/adaptive-log-parser/`

சுயமாகப் பரிணமிக்கக்கூடிய ஒரு பதிவு பாகுபடுத்தல் அமைப்பு: பாகுபடுத்த முடியாத புதிய வடிவமைப்பை எதிர்கொள்ளும்போது பிழை எழுப்புவதற்குப் பதிலாக, தோல்வியுற்ற மாதிரியையும் பிழைச் செய்தியையும் குறியீடு உருவாக்க ஏஜென்டிடம் கொடுத்து `parse` செயல்பாட்டை உருவாக்கச் செய்கிறது; தானியங்குச் சோதனை வெற்றியடைந்த பிறகு, அதைச் சூடாகப் புதுப்பித்து (hot update) பாகுபடுத்தல் பொறியில் பதிவு செய்கிறது — முழுச் செயல்முறையிலும் மனிதத் தலையீடு தேவையில்லை.

**முக்கிய கருத்துக்கள்**: அமைப்பு இணைப்பியாகக் குறியீடு, சுய-குணமடையும் மூடிய சுழற்சி, குறியீட்டின் சூடான புதுப்பிப்பு, தானியங்குச் சோதனை

### log-diagnosis - உற்பத்தி பதிவு நுண்ணறிவுக் கண்டறிதல் அமைப்பு
`chapter5/log-diagnosis/`

கண்டறிதல் ஏஜென்ட் உற்பத்தித் தடப் பதிவுகள் (trace logs), கட்டமைப்பு ஆவணங்கள் மற்றும் PRD-களைப் படித்து, பிரச்சினைகளையும் மூல காரணங்களையும் தானாகக் கண்டறிந்து, கட்டமைக்கப்பட்ட அறிக்கைகள் மற்றும் பின்னடைவுச் சோதனை நிகழ்வுகளை உருவாக்கி, மீள்செயல் கட்டமைப்பு (replay framework) மூலம் உண்மையான செயலாக்கச் சரிபார்ப்பை நடத்தி, (mock) MCP வழியாக GitHub-உடன் இணைந்து Issue உருவாக்குகிறது.

**முக்கிய கருத்துக்கள்**: தடக் கண்டறிதல், மூல காரணக் கண்டறிதல், பின்னடைவுச் சோதனை உருவாக்கம், மீள்செயல் சரிபார்ப்பு

### dynamic-form - நோக்கம் தெளிவுபடுத்தும் மாறும் படிவ அமைப்பு
`chapter5/dynamic-form/`

தகவல் முழுமையற்ற கோரிக்கையை எதிர்கொள்ளும்போது, ஏஜென்ட் ஒவ்வொன்றாக வினவாமல், படிநிலைத் தர்க்கம் (cascading logic) கொண்ட சுய-உள்ளடக்கிய HTML படிவத்தை மாறும் முறையில் உருவாக்கி, பயனர் ஒரே முறையில் முழுமைப்படுத்த அனுமதிக்கிறது; முன்னணி (frontend) படிவத்தை JSON-ஆகத் தொகுத்து ஏஜென்டிடம் திருப்பிக் கொடுத்துப் பணியைத் தொடர்கிறது.

**முக்கிய கருத்துக்கள்**: குறியீடு உருவாக்கம், நோக்கம் தெளிவுபடுத்தல், மாறும் படிவம், படிநிலைத் தர்க்கம்

சீன இயற்கை மொழி வினவல்களை SQL-ஆக மாற்றித் தரவுத்தளத்தில் இயக்கி, முடிவு அட்டவணையை நேரடியாகக் காட்டுகிறது; மையம் கலைப்பொருள் முறை (artifact pattern): LLM SQL கலைப்பொருளை மட்டுமே உருவாக்குகிறது, தரவைத் தானே நகர்த்துவதில்லை — இது token-ஐயும் மிச்சப்படுத்துகிறது, கைமுறைக் கணக்கீட்டுப் பிழைகளையும் தவிர்க்கிறது; பல்லாயிரக்கணக்கான வரிகள் கொண்ட முடிவுகளையும் நொடிகளில் திருப்பித் தர முடியும்.

**முக்கிய கருத்துக்கள்**: NL2SQL, கலைப்பொருள் முறை, தரவுத்தளச் செயலாக்கம், செலவு மற்றும் துல்லியத்தன்மை

**முக்கிய கருத்துக்கள்**: குறியீடு மாற்றம், முன்னணித் தனிப்பயனாக்கம், சூடான ஏற்றம், பல-சுற்று மீள்செயல்

### model-benchmark - பல்-பரிமாண மாதிரி செயல்திறன் தரநிர்ணயம்

**முக்கிய கருத்துக்கள்**: TTFT, தாமத குவான்டைல்கள், செயலாக்க விகிதம் (throughput), இணையான அழுத்தச் சோதனை, மாதிரி தேர்வு

**முக்கிய கருத்துக்கள்**: தூண்டுதல் தானியங்கி மேம்படுத்தல், மனிதக் கருத்து, Coding Agent, மூடிய சுழற்சி மதிப்பீடு

"120+ கருவி schema-களை முழுமையாக உட்செலுத்துதல்" மற்றும் "தேவைப்படும்போது முன்னெச்சரிக்கையுடன் கண்டுபிடித்தல்" என்ற இரண்டு முன்னுதாரணங்களை ஒப்பிடுகிறது: பிந்தையது system-இல் சில அடிப்படைக் கருவிகள் + ஒரு `discover_tools` மெட்டா-கருவியை மட்டுமே வைத்திருந்து, உட்பொதிவு ஒற்றுமையைப் பயன்படுத்தி கருவி நூலகத்திலிருந்து மிகவும் தொடர்புடைய 3-5 சிறப்புக் கருவிகளை மீட்டெடுக்கிறது—இது டோக்கனை மிச்சப்படுத்துவதுடன், மிக நீண்ட கருவிப் பட்டியலில் மாதிரி பொதுநோக்குக் கருவிகளை தவறாகத் தேர்ந்தெடுப்பது/தவறாகப் பயன்படுத்துவதையும் தவிர்க்கிறது.

**முக்கிய கருத்துக்கள்**: முன்னெச்சரிக்கை கருவி கண்டுபிடிப்பு, உட்பொதிவு மீட்டெடுப்பு, டோக்கன் மேம்படுத்தல், கட்டளை பின்பற்றுதல்

### self-evolving-tools - இணையத்திலிருந்து கருவிகளைத் தேடி சுய-பரிணாமம் அடைதல்
`chapter8/self-evolving-tools/`

Alita பாணி "குறைந்தபட்ச முன்வரையறை, அதிகபட்ச சுய-பரிணாமம்": ஏஜென்ட் எந்தத் துறைச் கருவிகளையும் முன்கூட்டியே கொண்டிருக்காமல், ஐந்து பொதுநோக்கு மெட்டா-கருவிகள் மட்டுமே கொண்டு, செய்யத் தெரியாத பணியை எதிர்கொள்ளும்போது தானே இணையத்தில் திறந்த மூல நூலகங்கள்/API-களைத் தேடி, ஆவணங்களைப் படித்து, மணற்பெட்டியில் (sandbox) சோதித்து, சாத்தியமான தீர்வுகளைப் புதிய கருவிகளாக உறையிட்டு கருவி நூலகத்தில் சேமித்து மறுபயன்படுத்துகிறது—முழுச் செயல்முறையிலும் ஊகக் கட்டுப்பாட்டை (hallucination control) வலியுறுத்துகிறது.

**முக்கிய கருத்துக்கள்**: சுய-பரிணாமம், கருவி படைப்பு, கருவி மறுபயன்பாடு, ஊகக் கட்டுப்பாடு

### self-evolution-eval - சுய-பரிணாம ஏஜென்ட் மதிப்பீட்டுத் தரவுத்தொகுப்பு
`chapter8/self-evolution-eval/`

ஏஜென்ட்டின் "சுய-பரிணாம" திறனை (தானே கருவிகளைக் கண்டுபிடித்து, உருவாக்கி, மறுபயன்படுத்துதல்) மதிப்பீடு செய்ய வடிவமைக்கப்பட்ட சிறப்புத் தரவுத்தொகுப்பு மற்றும் சரிபார்ப்பு முறை: 20 பல்துறை பணிகள் (கருவி பெயர்களைக் குறிப்பிடாமல்) + நான்கு-அடுக்குப் படிநிலைச் சரிபார்ப்பு harness + கட்டுப்படுத்தக்கூடிய குறிப்பு ஏஜென்ட்—"முடிவு சரியா தவறா" என்பதைத் தாண்டி, கண்டுபிடிப்பு, படைப்பு மற்றும் மறுபயன்பாட்டுத் தரத்தை ஆராய்கிறது.

**முக்கிய கருத்துக்கள்**: மதிப்பீட்டுத் தரவுத்தொகுப்பு வடிவமைப்பு, படிநிலைச் சரிபார்ப்பு, கருவி மறுபயன்பாட்டு அளவீடு, சுய-பரிணாமம்

### live-audio - நிகழ்நேர குரல் உரையாடல்
`chapter9/live-audio/`

நிகழ்நேர குரல் அரட்டை டெமோ, பேச்சு-முதல்-உரை, AI உரையாடல் மற்றும் உரை-முதல்-பேச்சு செயல்பாடுகளை ஒருங்கிணைக்கிறது. பல AI சேவை வழங்குநர்களை (OpenAI, OpenRouter, ARK, Siliconflow) ஆதரித்து, குறைந்த தாமத உரையாடல் அனுபவத்தை வழங்குகிறது.

**முக்கிய அம்சங்கள்**:
- நிகழ்நேர குரல் உள்ளீடு மற்றும் VAD (Voice Activity Detection)
- பல வழங்குநர் ஆதரவு: ASR (OpenAI Whisper, SenseVoice), LLM (GPT-4o, Gemini, Doubao), TTS (Fish Audio)
- WebSocket நிகழ்நேர தொடர்பு, குறைந்த தாமத ஆடியோ ஸ்ட்ரீமிங்
- நிகழ்நேர தாமதக் கண்காணிப்பு மற்றும் பதிவு செய்தல்

**முக்கிய கருத்துக்கள்**: பேச்சு அங்கீகாரம், நிகழ்நேர உரையாடல், TTS, WebSocket, பல-வழங்குநர் கட்டமைப்பு

### browser-use - உலாவி தானியக்க ஏஜென்ட் (Computer Use)
`chapter9/browser-use/`

Browser-Use என்பது ஒரு சக்திவாய்ந்த உலாவி தானியக்கக் கட்டமைப்பு ஆகும், இது LLM-கள் உலாவியைக் கட்டுப்படுத்தி சிக்கலான பணிகளை முடிக்க உதவுகிறது. படிவம் நிரப்புதல், வலைப்பக்க வழிசெலுத்தல், தரவுப் பிரித்தெடுப்பு போன்ற சூழ்நிலைகளை ஆதரிக்கிறது, மேலும் GUI தானியக்கத்தின் (Computer Use) ஒரு எடுத்துக்காட்டுச் செயலாக்கமாகும்.

**முக்கிய அம்சங்கள்**:
- LLM இயக்கும் உலாவி தானியக்கம்
- பல LLM-களை ஆதரித்தல் (ChatBrowserUse, OpenAI, Google, உள்ளூர் மாதிரிகள்)
- தனிப்பயன் கருவி நீட்டிப்புகள், அங்கீகாரக் கையாளுதல்
- மணற்பெட்டி (sandbox) வரிசைப்படுத்தல் ஆதரவு, மேகக்கணி சேவை ஒருங்கிணைப்பு

**முக்கிய கருத்துக்கள்**: உலாவி தானியக்கம், Computer Use, காட்சிப் புரிதல், கருவி நீட்டிப்பு

### phone-agent - தொலைபேசி ஏஜென்ட்
`chapter9/phone-agent/`

குரல் ஏஜென்ட் "பயனருக்குப் பதிலாக வெளி உலகத்துடன் தொலைபேசி மூலம் தொடர்பு கொள்வதை" விளக்குகிறது: மேல் அடுக்கில் ஒரு நிலையான ReAct ஏஜென்ட் உள்ளது; இயற்கை மொழிப் பணியைப் பெற்றவுடன், எண் மற்றும் அழைப்பு இலக்கைத் தானே தெளிவுபடுத்தி, `make_phone_call` கருவியை (தொலைபேசி குரல் API அமுங்கம் அடிப்படையில்) அழைத்து முழு அழைப்பையும் முடித்து, கட்டமைக்கப்பட்ட அழைப்புப் பதிவுகளைப் படித்து, தேவைப்பட்டால் வினாடி மீண்டும் அழைத்து, இறுதியாக பயனருக்கு அறிக்கை செய்கிறது.

**முக்கிய கருத்துக்கள்**: குரல் ஏஜென்ட், தொலைபேசி இடைவினை, ReAct, கருவி அமுங்கம்

### end-to-end-speech - இறுதி-முதல்-இறுதி குரல் சிந்தனை vs அடுக்குக் குழாய்
`chapter9/end-to-end-speech/`

Step-Audio R1-இன் இறுதி-முதல்-இறுதி குரல் சிந்தனை முன்னுதாரணத்துடன் (ஒற்றை மாதிரி "கேள் → சிந்தி → பேசு") ஒத்திருக்கிறது: "குரல் உள்ளீடு → சிந்தனை → குரல் வெளியீடு" மூடிய சுழற்சியை இயக்கி, ASR→LLM→TTS அடுக்கு முன்னுதாரணத்துடன் தாமதம் மற்றும் துணைமொழித் தகவல் (உணர்ச்சி/தொனி/பேச்சு வேகம்) இழப்பு வேறுபாட்டை நேரடியாக ஒப்பிடுகிறது.

**முக்கிய கருத்துக்கள்**: இறுதி-முதல்-இறுதி குரல், அடுக்கு ஒப்பீடு, துணைமொழித் தகவல், சிந்தித்துக்கொண்டே பேசுதல்

### streaming-speech - உருவகப்படுத்தப்பட்ட ஸ்ட்ரீமிங் குரல் உணர்தல்
`chapter9/streaming-speech/`

ஸ்ட்ரீமிங் குரல் உணர்தலின் முக்கிய பரிமாற்றத்தை விளக்குகிறது: தொடர்ச்சியான ஆடியோவை அதிகரிக்கும் நீளத் தொகுதிகளாகப் பிரித்து ASR-க்கு ஊட்டி, ஒவ்வொரு சிறு பகுதி கிடைத்தவுடன் "தற்போதைய பகுதி அங்கீகார முடிவை" உருவாக்கி, மிகக் குறைந்த முதல்-பாக்கெட் தாமதத்துடன் (first-packet latency) உரையை விரைவில் வெளியிடுகிறது; இதன் விலை, பின்னர் வரும் வாக்கியச் சூழல் இல்லாததால் ஆரம்பத் தொகுதிகள் தவறாக இருக்கலாம், ஆடியோ குவியும்போது படிப்படியாக ஒருங்கமைகிறது—"முழு வாக்கியம் வரும் வரை காத்திருந்து அங்கீகரித்தல்" என்ற உயர் துல்லியம்/உயர் தாமத அணுகுமுறைக்கு மாறானதாக உள்ளது.

**முக்கிய கருத்துக்கள்**: ஸ்ட்ரீமிங் உணர்தல், தொகுதி வாரியான அங்கீகாரம், முதல்-பாக்கெட் தாமதம், முன்கூட்டிய முடிவெடுப்பின் விலை

### controllable-tts - கட்டுப்பாட்டுக் குறிகளால் இயக்கப்படும் கட்டுப்படுத்தக்கூடிய TTS
`chapter9/controllable-tts/`

முதன்மை LLM-இன் வெளியீட்டில் கட்டுப்பாட்டுக் குறிகளை (உணர்ச்சி/பேச்சு வேகம்/பாணி/இடைநிறுத்தம்/சிரிப்பு) இணைத்து, செயலாக்க அடுக்கு அந்தக் குறிகளைப் பாகுபடுத்தி குறிப்புக் குரல் நூலகத்தில் உள்ள தொடர்புடைய பாணிச் சுயவிவரங்களுடன் பொருத்தி பேச்சைத் தொகுக்கிறது—"எங்கே இடைநிறுத்தம் செய்ய வேண்டும், எந்த தொனியைப் பயன்படுத்த வேண்டும்" என்ற முடிவுகளை LLM-இடம் ஒப்படைத்து, ஒரே உரையை வெவ்வேறு பாணிகள் மற்றும் உணர்ச்சிகளுடன் தொகுக்க முடியும்.

**முக்கிய கருத்துக்கள்**: கட்டுப்படுத்தக்கூடிய TTS, கட்டுப்பாட்டுக் குறிகள், குறிப்புக் குரல் நூலகம், ஒலியியைபுக் கட்டுப்பாடு (prosody control)

### use-computer-while-calling - இரு-ஏஜென்ட் கட்டமைப்பு
`chapter10/use-computer-while-calling/` (📖 முழுமையான குறியீடு [19PINE-AI/TalkAct](https://github.com/19PINE-AI/TalkAct) எனத் தனியாகப் பிரிக்கப்பட்டது, இந்த அடைவு விளக்க ஆவணத்தை மட்டுமே கொண்டுள்ளது)

தொலைபேசி அழைப்பு ஏஜென்ட் மற்றும் கணினி பயன்பாட்டு ஏஜென்ட் இடையேயான இரு-ஏஜென்ட் ஒத்துழைப்புக் கட்டமைப்பைச் செயல்படுத்துகிறது. இரண்டு ஏஜெண்டுகளும் ஒருங்கிணைப்பாளர் இல்லாமல் WebSocket மூலம் நேரடியாகத் தொடர்பு கொள்கின்றன. தொலைபேசி ஏஜென்ட் குரல் தொடர்பைக் கையாள்கிறது, கணினி ஏஜென்ட் உலாவி தன்னியக்கத்தைச் செயல்படுத்துகிறது—குரல் மற்றும் வலைப்பக்கச் செயல்பாடுகள் தேவைப்படும் சிக்கலான பணிகளை இணையாகச் செய்து முடிக்கின்றன.

**முக்கிய அம்சங்கள்**:
- நேரடி ஏஜென்ட்-இடைத் தொடர்பு (ஒருங்கிணைப்பாளர் இல்லை)
- செய்தி அனுப்புதலுக்குத் தரநிலையான கருவி அழைப்புகள்
- இணையான செயல்பாடு: குரல் உரையாடல் + உலாவி தன்னியக்கம்
- எளிய JSON செய்தி நெறிமுறை

**கட்டமைப்புக் கூறுகள்**:
- Phone Call Agent (Node.js): குரல் I/O, ASR/TTS, LLM உரையாடல்
- Computer Use Agent (Python): உலாவி தன்னியக்கம், browser-use, வலைப்பக்கச் சேகரிப்பு (web scraping)
- WebSocket தொடர்பு: ஏஜெண்டுகளுக்கு இடையே நேரடி செய்தி அனுப்புதல்

**முக்கிய கருத்துக்கள்**: பல-ஏஜென்ட் ஒத்துழைப்பு, ஏஜென்ட்-இடைத் தொடர்பு, இணையான பணிச் செயலாக்கம், குரல் + உலாவி ஒருங்கிணைப்பு

### staged-system-prompt - செயலாக்க நிலைகளுக்கேற்ப கணினி வழிகாட்டியை மாற்றுதல்
`chapter10/staged-system-prompt/`

அதே Coding Agent ஒரு பணியின் வெவ்வேறு செயலாக்க நிலைகளில் (தேவைகள் தெளிவுபடுத்தல் → குறியீடு செயலாக்கம் → குறியீடு மதிப்பாய்வு) வெவ்வேறு கணினி வழிகாட்டிகள் மற்றும் கருவித் தொகுப்புகளை ஏற்றுகிறது; இதன் மூலம் ஒரே உரையாடலில் வெவ்வேறு பாத்திரங்களை ஏற்று வெவ்வேறு நடத்தைகளை வெளிப்படுத்துகிறது, அதே நேரத்தில் உரையாடல் வரலாறு மற்றும் பணி நிலை நிலைகளுக்கிடையே தொடர்ச்சியாகப் பகிரப்படுகின்றன. மதிப்பாய்வில் தேர்ச்சி பெறாவிட்டால், செயலாக்க நிலைக்குத் திரும்பவும் முடியும்.

**முக்கிய கருத்துக்கள்**: நிலைப்படுத்தப்பட்ட வழிகாட்டிகள், பாத்திர மாற்றம், பகிரப்பட்ட சூழல், நிலை பாதை (stage pipeline)

### multi-role-transfer - பல-பாத்திர மாற்றம் மற்றும் தன்னாட்சி ஒப்படைப்பு
`chapter10/multi-role-transfer/`

பகிரப்பட்ட சூழலின் கீழ் சங்கிலி ஒப்படைப்பை (handoff) நிரூபிக்கிறது: ஒரே அமர்வில் பல சிறப்புப் பாத்திர ஏஜெண்டுகள் உள்ளன, ஒவ்வொன்றும் சொந்த கணினி வழிகாட்டி மற்றும் அர்ப்பணிக்கப்பட்ட கருவித் தொகுப்பைக் கொண்டுள்ளன. `transfer_to_agent` கருவி மூலம், பணி முன்னேற்றத்தின் அடிப்படையில் எந்தப் பாத்திரத்திற்கு மாற வேண்டும் என்பதை ஏஜென்ட் தானாகவே முடிவு செய்கிறது. அதே உரையாடல் வரலாற்றைப் பகிர்வதால், ஒப்படைப்பின் போது முழுமையான சூழல் இயற்கையாகவே பாதுகாக்கப்படுகிறது.

**முக்கிய கருத்துக்கள்**: பாத்திர ஒப்படைப்பு, handoff, பகிரப்பட்ட சூழல், தன்னாட்சி மாற்றம்

### book-translation - புத்தக மொழிபெயர்ப்பு ஏஜென்ட் (மேலாளர் முறை)
`chapter10/book-translation/`

மேலாளர் முறையை (Orchestration) பயன்படுத்தி நீண்ட ஆவண மொழிபெயர்ப்பைச் சொற்களஞ்சியம்/மொழிபெயர்ப்பு/சரிபார்ப்பு போன்ற அர்ப்பணிப்பு ஏஜெண்டுகளுக்குப் பிரித்துக் கொடுக்கிறது: Manager பணிகள், திட்டங்கள், அழைப்புப் பதிவுகள் மற்றும் கோப்புக் குறியீடுகளை மட்டுமே சேமிக்கிறது, முழுமையான மொழிபெயர்ப்பு முழுவதும் வட்டில் எழுதப்படுவதால் சூழல் அடிப்படையில் மாறாமல் இருக்கிறது. மேலும் ஒற்றை-ஏஜென்ட் அணுகுமுறையுடன் ஒப்பிட்டு, சூழல் வீக்கத்தை (context explosion) எவ்வாறுக் கட்டுப்படுத்துவது, பகிரப்பட்ட சொற்களஞ்சியத்தால் புத்தகம் முழுவதும் நிலைத்தன்மையை எவ்வாறு உறுதி செய்வது என்பதை உண்மையான token எண்ணிக்கைகளுடன் விளக்குகிறது.

**முக்கிய கருத்துக்கள்**: மேலாளர் முறை, சூழல் தனிமைப்படுத்தல், சூழல் வீக்கக் கட்டுப்பாடு, பகிரப்பட்ட சொற்களஞ்சியம்

### parallel-web-research - இணையான பல-மூலத் தகவல் சேகரிப்பு ஏஜென்ட்
`chapter10/parallel-web-research/`

பல ஒருமுக ஏஜெண்டுகளின் இணையான தேடல் + மைய ஒருங்கிணைப்பை நிரூபிக்கிறது: முதன்மை ஒருங்கிணைப்பாளர் ஒரே நேரத்தில் N துணை ஏஜெண்டுகளைத் தொடங்குகிறது, ஒவ்வொன்றும் ஒரு மூலத்தை அணுகி பதிலைத் தேடுகின்றன; ஏதேனும் ஒன்று இலக்கை அடைந்தவுடன், மற்றவை உடனடியாக மென்மையாக நிறுத்தப்படுகின்றன. செய்தி பேருந்து, இணையான அனுப்பல், நிகழ்நேரக் கண்காணிப்பு, அடுக்கு நிறுத்தம் (cascading termination) மற்றும் போட்டி நிலைக் கையாளுதல் (race condition handling) அனைத்தும் உண்மையான செயலாக்கங்களாகும் (உண்மையான உலாவிக்குப் பதிலாகக் கட்டுப்படுத்தக்கூடிய உருவகப்படுத்தப்பட்ட தகவல் மூலங்களைப் பயன்படுத்துகிறது).

**முக்கிய கருத்துக்கள்**: இணையான ஏஜெண்டுகள், மைய ஒருங்கிணைப்பு, செய்தி பேருந்து, அடுக்கு நிறுத்தம்

### voice-werewolf - குரல் வேர்வுல்ஃப் ஏஜென்ட் அமைப்பு
`chapter10/voice-werewolf/`

பல-ஏஜென்ட் வேர்வுல்ஃப் (Werewolf) விளையாட்டைப் பயன்படுத்தி "சூழல் பகிரப்படாத" நிலையில் தகவல் அனுமதிக் கட்டுப்பாட்டை நிரூபிக்கிறது: ஒவ்வொரு வீரரும் ஒரு சுயாதீன LLM ஏஜென்ட் ஆகும், கண்டிப்பாகத் தனிமைப்படுத்தப்பட்ட தனிப்பட்ட சூழலைப் பராமரிக்கிறது. குறியீட்டால் இயக்கப்படும் தீர்மானமான நீதிபதி (deterministic judge) ஒவ்வொரு தகவலும் எந்த வீரர்களின் சூழலுக்கு வழங்கப்பட வேண்டும் என்பதைத் தீர்மானித்து, தணிக்கைக்காகப் பதிவு செய்கிறார்; விளையாட்டு முடிவில் தனிமைப்படுத்தல் சரியாக உள்ளதா என்பது தானாகவே சரிபார்க்கப்படுகிறது. குரல் ஒரு விருப்ப மேம்பாடாகும்.

**முக்கிய கருத்துக்கள்**: தகவல் சமச்சீரின்மை, தனிப்பட்ட சூழல் தனிமைப்படுத்தல், நீதிபதி ஒருங்கிணைப்பு, தணிக்கைச் சரிபார்ப்பு

## 📖 கற்றல் பரிந்துரைகள்

### மையக் கருத்து: ஏஜென்ட் = மாதிரி + சூழல் + கருவிகள்

இந்தப் புத்தகத்தின் மையக் கட்டமைப்பு **ஏஜென்ட் = மாதிரி + சூழல் + கருவிகள்**; இந்த மூன்று கூறுகள் ஒன்றுடன் ஒன்று இணைந்து செயல்பட்டு, ஏஜென்டின் அறிவார்ந்த நடத்தையை இணைந்து செயல்படுத்துகின்றன:

- **மாதிரி (Model)**: ஏஜென்டின் மூளை; புரிதல், பகுத்தறிவு மற்றும் முடிவெடுக்கும் திறன்களை வழங்குகிறது
- **சூழல் (Context)**: ஏஜென்டின் இயக்க முறைமை; கணினி வழிமுறைகள், உரையாடல் வரலாறு, பகுத்தறிவு செயல்முறைகள், கருவி தொடர்புப் பதிவுகள் போன்றவற்றைக் கொண்டுள்ளது
- **கருவிகள் (Tools)**: ஏஜென்டின் கைகள்; சூழலை உணரவும், செயல்பாடுகளைச் செய்யவும், வெளி உலகத்துடன் தொடர்பு கொள்ளவும் ஏஜென்டை முடுக்குகிறது

### கற்றல் பாதை

கற்றல் பாதை புத்தகத்தின் அத்தியாயங்களுடன் ஒன்றுக்கொன்று ஒத்துப்போகிறது, மூன்று தூண்களைச் சுற்றி அடுக்கடுக்காக விரிவடைகிறது:

- **அத்தியாயம் 1 · அடித்தளம்**: ஏஜென்ட் அமைப்புகள் பற்றிய முழுமையான அறிவாற்றல் கட்டமைப்பை உருவாக்குதல்—RL இல் ஏஜென்டின் வரையறையைப் புரிந்துகொள்ளுதல், பாரம்பரிய RL மற்றும் LLM+RL முன்னுதாரணங்களுக்கிடையேயான மாதிரி திறன் (sample efficiency) வேறுபாடுகளை ஒப்பிடுதல், "மாதிரியே ஏஜென்டாக" என்ற புதிய முன்னுதாரணத்தைப் புரிந்துகொள்ளுதல், மற்றும் **ஏஜென்ட் = மாதிரி + சூழல் + கருவிகள்** என்ற மையக் கட்டமைப்பை மாஸ்டர் செய்தல். **முக்கிய நுண்ணறிவு**: முன்னறிவின் (prior knowledge) முக்கியத்துவம் வழிமுறைகளையும் சூழல்களையும் விட மேலானது.

- **அத்தியாயங்கள் 2–3 · சூழல்**: சூழல் என்பது ஏஜென்டின் இயக்க முறைமை. அத்தியாயம் 2 கணினி வழிகாட்டிகள், KV Cache-நட்பு வடிவமைப்பு, சூழல் சுருக்கம் மற்றும் வழிகாட்டி பொறியியல் அப்லேஷன் ஆகியவற்றை உள்ளடக்குகிறது; அத்தியாயம் 3 பயனர் நினைவகம், அடர்த்தியான/அரிதான/கலப்பு மீட்டெடுப்பு, Agentic RAG, சூழல்-உணர்ந்த மீட்டெடுப்பு (contextual retrieval) மற்றும் கட்டமைக்கப்பட்ட அறிவுப் பிரித்தெடுப்பு ஆகியவற்றை உள்ளடக்குகிறது. **முக்கிய நுண்ணறிவு**: முழுமையான சூழலில் கணினி வழிமுறைகள், உரையாடல் வரலாறு, பகுத்தறிவு செயல்முறைகள், கருவி தொடர்புப் பதிவுகள், பயனர் நினைவகம் மற்றும் வெளிப்புற அறிவு ஆகியவை அடங்கும்.

- **அத்தியாயங்கள் 4–5 · கருவிகள்**: கருவிகள் ஏஜென்ட் உலகத்துடன் தொடர்பு கொள்ளும் பாலமாகும். அத்தியாயம் 4 உணர்தல்/செயலாக்கம்/ஒத்துழைப்பு என்ற மூன்று வகை MCP கருவிகள், நிகழ்வுத் தூண்டுதல் மற்றும் ஒத்திசைவற்ற கட்டமைப்பு ஆகியவற்றை உள்ளடக்குகிறது; அத்தியாயம் 5 உற்பத்தி-தர Coding Agent இன் முழுமையான செயலாக்கத்தை ஆழமாக ஆராய்கிறது. **முக்கிய நுண்ணறிவு**: கருவி வடிவமைப்பு பொதுமைப்படுத்தப்பட வேண்டும் (குறியீடு மொழிபெயர்ப்பாளர் கால்குலேட்டரை விடச் சிறந்தது); குறியீடு என்பது புதிய கருவிகளை உருவாக்கக்கூடிய மெட்டா-திறன் ஆகும்.

- **அத்தியாயங்கள் 6–7 · மாதிரி**: நுண்ணறிவை எவ்வாறு அளவிடுவது மற்றும் பெரிதாக்குவது. அத்தியாயம் 6 Terminal-Bench, SWE-bench, GAIA, OSWorld, Tau2-Bench போன்ற மதிப்பீட்டுத் தரநிர்ணயங்களை உள்ளடக்குகிறது; அத்தியாயம் 7 SFT, RL, RLHF, மாதிரி திறன் போன்ற பிந்தைய-பயிற்சி நுட்பங்களை உள்ளடக்குகிறது. **முக்கிய நுண்ணறிவு**: "மாதிரியை மீண்டும் யோசிக்கச் செய்வதை" விட ஒரு சுயாதீன சரிபார்ப்புச் சமிக்ஞை மிகவும் நம்பகமானது; "மாதிரியே ஏஜென்டாக" RL மூலம் கருவி அழைப்புகளை உள்ளார்ந்த திறன்களாக உள்வாங்குகிறது.

- **அத்தியாயம் 8 · சுய-பரிணாமம்**: எடைகளை மாற்றாமல் அனுபவத்திலிருந்து ஏஜென்ட் வளரச் செய்தல்—அனுபவக் கற்றல், பணிப்பாய்வுகளைக் கருவிகளாக வெளிப்புறமயமாக்குதல், வழிகாட்டிகள் மற்றும் கவனிப்புகளை அளவுருக்களில் வடிகட்டுதல். **முக்கிய நுண்ணறிவு**: அனுபவத்திலிருந்து கற்றல் என்பது ஒரு ஏஜென்ட் "புத்திசாலி" என்பதிலிருந்து "திறமையான" நிலைக்குச் செல்வதற்கான திறவுகோலாகும்.

- **அத்தியாயங்கள் 9–10 · விரிவாக்கம் மற்றும் ஒத்துழைப்பு**: அத்தியாயம் 9 உணர்தல் மற்றும் செயலை உரையிலிருந்து குரல், GUI மற்றும் இயற்பியல் உலகத்திற்கு விரிவுபடுத்துகிறது; அத்தியாயம் 10 பல-ஏஜென்ட் பணிப் பிரிவு ஒத்துழைப்பின் மூலம் சிக்கலான பணிகளைக் கையாள்கிறது. **முக்கிய நுண்ணறிவு**: பல-ஏஜென்ட் அமைப்பின் ஒவ்வொரு வடிவமைப்பு முடிவுக்கும் ஒற்றை ஏஜென்டின் மூன்று கூறுகளில் ஒரு ஒத்துப்போவதைக் காணலாம்.

### சிரம நிலை வகைப்பாடு

- **தொடக்க நிலை** (அத்தியாயங்கள் 1–2): ஆரம்பநிலையாளர்களுக்கு ஏற்றது, அடிப்படைக் கருத்துக்களைப் புரிந்துகொள்ளுதல்
- **இடைநிலை** (அத்தியாயங்கள் 3–4): ஓரளவு நிரலாக்க அடித்தளம் தேவை, அமைப்பு ஒருங்கிணைப்பு உள்ளடக்கியது
- **மேம்பட்ட நிலை** (அத்தியாயங்கள் 5–6): வலுவான நிரலாக்கத் திறன் தேவை, சிக்கலான அமைப்பு வடிவமைப்பு உள்ளடக்கியது
- **நிபுணர் நிலை** (அத்தியாயங்கள் 7–8): ஆழ்ந்த கற்றல் (deep learning) மற்றும் பயிற்சி/சுய-பரிணாம அனுபவம் தேவை
- **பயன்பாட்டு நிலை** (அத்தியாயங்கள் 9–10): முன்னர் கற்றவற்றை ஒருங்கிணைத்துப் பயன்படுத்தி, நடைமுறைப் பயன்பாடுகளை உருவாக்குதல்

### நடைமுறைப் பரிந்துரைகள்

1. **நடைமுறைப் பயிற்சி**: ஒவ்வொரு திட்டமும் சுயாதீனமாக இயக்கக்கூடியதாக வடிவமைக்கப்பட்டுள்ளது; நீங்களே குறியீட்டை இயக்கி மாற்றிப் பார்க்க பரிந்துரைக்கப்படுகிறது
2. **புத்தகத்துடன் இணைத்துப் படிக்கவும்**: இந்தக் களஞ்சியத்தின் [`book/`](book/) அடைவில் உள்ள கையேட்டின் தொடர்புடைய அத்தியாயங்களுடன் சேர்த்துப் படித்து, கோட்பாடு மற்றும் நடைமுறையின் இணைப்பைப் புரிந்துகொள்ளவும்
3. **சோதனை ஒப்பீடு**: பல திட்டங்கள் அப்லேஷன் ஆய்வுகள் மற்றும் ஒப்பீட்டுச் சோதனைகளைக் கொண்டுள்ளன; ஒப்பீட்டின் மூலம் புரிதலை ஆழப்படுத்தவும்
4. **படிப்படியான கற்றல்**: எளிய திட்டங்களிலிருந்து தொடங்கி, படிப்படியாகச் சிக்கலான அமைப்புகளுக்குச் செல்லவும்
5. **நெறிமுறைகளில் கவனம் செலுத்தவும்**: அத்தியாயம் 4 இல் உள்ள MCP சேவையகத் திட்டங்கள் தரநிலையான கருவி நெறிமுறைகளை நிரூபிக்கின்றன, இவை அளவிடக்கூடிய ஏஜெண்டுகளை உருவாக்குவதற்கான திறவுகோலாகும்

## 🤝 பங்களிப்பு

இப்புத்தகமும் அதனுடன் வரும் குறியீடும் முழுமையாகத் திறந்த மூலமாகும்; சமூகம் Pull Request மூலம் இணைந்து பங்களிப்பதை மிகவும் வரவேற்கிறோம். பின்வரும் வகையான பங்களிப்புகளை நாங்கள் மகிழ்ச்சியுடன் வரவேற்கிறோம்:

1. **புத்தக உள்ளடக்க மேம்பாடுகள்**: பிழைத் திருத்தங்கள், துணைச் சேர்க்கைகள், இன்னும் தெளிவான வெளிப்பாடுகள், அல்லது புதிய முன்னணி முன்னேற்றங்களைச் சேர்த்தல் (தமிழ் உள்ளடக்கத்திற்கு `book-ta/chapter*.ta.md` ஐப் பார்க்கவும்)
2. **குறியீட்டு மேம்பாடுகள் மற்றும் Bug திருத்தங்கள்**: இணைந்து வரும் திட்டங்களை மேலும் உறுதியாகவும், பயன்பட எளிதாகவும், உற்பத்தி நடைமுறைக்கு நெருக்கமாகவும் மாற்றுதல்
3. **புதிய நடைமுறைத் திட்டங்கள்**: ஏதாவது ஒரு சோதனைக்கு சிறந்த செயலாக்கத்தைச் சேர்த்தல்/மாற்றுதல், அல்லது முற்றிலும் புதிய எடுத்துக்காட்டுத் திட்டத்தைப் பங்களித்தல்
4. **புத்தக விளக்கப்படங்களின் வடிவமைப்பு மேம்பாடுகள்**: `book-ta/images/` இல் உள்ள விளக்கப்படங்களை வடிவமைப்பில் மேலும் தெளிவாகவும் அழகாகவும் மாற்றுதல் (விளக்கப்படங்கள் `book/gen_*_figs.py` ஆல் உருவாக்கப்படுகின்றன)
5. **புதிய மொழி மொழிபெயர்ப்புப் பதிப்புகள்**: இப்புத்தகத்தை மேலும் பல மொழிகளில் மொழிபெயர்க்க வரவேற்கிறோம் — ஆங்கில (`book-en/`), தமிழ் (`book-ta/`) மற்றும் வியட்நாமிய (`book-vi/`) பதிப்புகள் எவ்வாறு ஒழுங்கமைக்கப்பட்டுள்ளன என்பதைப் பார்க்கவும்

சமர்ப்பிப்பதற்கு முன், தொடர்புடைய சோதனைகளை நீங்களே இயக்கி, மறுஉருவாக்கம் செய்ய முடிகிறது என உறுதிப்படுத்துவது நல்லது; உங்கள் யோசனைகளை விவாதிக்க முதலில் ஒரு issue திறப்பதையும் வரவேற்கிறோம்.
